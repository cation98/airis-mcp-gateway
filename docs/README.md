# Documentation Overview

The `docs/` tree is now organized into three buckets so you can find content without digging through root-level files.

| Folder | Purpose | Key Files |
|--------|---------|-----------|
| `guides/` | Maintained playbooks and how-tos. Update these when workflows change. | `mcp-installation.md`, `mcp-best-practices.md`, `augmentor-abi.md`, `augmentor-integration.md`, `testing-implementation-summary.md` |
| `research/` | Snapshot-style reports, measurement data, and historical analyses. Keep raw findings here so guides can stay concise. | `README.md`, `token-measurement-guide.md`, `PHASE2_IMPLEMENTATION_SUMMARY.md`, `mcp_installation_best_practices_20251018.md`, `mcp_servers_verification_20251018.md` |
| `memory/` | Team journaling context (PM briefs, next actions). Not versioned elsewhere. | `pm_context.md`, `next_actions.md` |

## Quick Links

- **Install the Gateway** → `guides/mcp-installation.md`
- **Decide which MCP servers to enable** → `guides/mcp-best-practices.md`
- **Augmentor ABI + integration plan** → `guides/augmentor-abi.md` and `guides/augmentor-integration.md`
- **Token measurement workflow** → `research/token-measurement-guide.md`

If you create a new document:
1. Decide whether it is an actionable guide or a research log.
2. Drop it into the corresponding folder.
3. Add it to the table or quick links above as needed.
